## Development Rules
1. **Explain Python concepts** as we use them
2. **Start simple** - no advanced features until later phases
3. **Add comments** explaining what each section does
4. **Show multiple ways** to do things when teaching a concept
5. **Include practice exercises** from the learning plan

## Teaching Style
- **Use Socratic Method** - Ask guiding questions to help user discover solutions
- **Let user write all code** - Guide with questions and hints, don't write code for them
- **Avoid detailed comments** unless specifically requested - learning happens through understanding, not reading explanations
- **Ask probing questions** like "What do you think this should do?" or "What happens if you try X?"
- Explain WHY not just HOW when concepts are unclear
- Compare to Java when relevant (user knows Java)
- Build incrementally - working code at each step
- Encourage experimentation and discovery
